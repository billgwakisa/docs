The x-bridge MCP server turns your coding assistant (Claude Code, Cursor, any MCP client) into an x-bridge integration expert. It scaffolds x-bridge **into your codebase** — a token client, the flow for your use case, and the gotchas baked in — so you ship the integration in minutes.

<Note>
By default it runs in **integrate mode**: it writes integration code into your project and never calls live x-bridge APIs on its own. (An optional execute mode lets the agent run sandbox calls — see the end.)
</Note>

## Install

<Warning>
Prototype — `@x-bridge/mcp` is not published to npm yet. For now, install from the repo. The `npx` one-liner below is the planned path.
</Warning>

```bash
# Planned (once published):
claude mcp add x-bridge -- npx -y @x-bridge/mcp

# Today (from the repo):
cd mcp && npm install && npm run build
BRIDGE_MODE=integrate claude mcp add x-bridge -- node /absolute/path/to/docs/mcp/dist/index.js
```

Integrate mode needs **no credentials** — it only writes code, it doesn't call the API.

### Cursor (`~/.cursor/mcp.json`)

```json
{
  "mcpServers": {
    "x-bridge": {
      "command": "node",
      "args": ["/absolute/path/to/docs/mcp/dist/index.js"],
      "env": { "BRIDGE_MODE": "integrate" }
    }
  }
}
```

## Use it

In your editor, run the **`integrate`** prompt, or just ask:

> *"Add x-bridge BNPL to this app."*  ·  *"Wire up x-bridge wallets in TypeScript."*

The agent will:
1. Read the integration rules (`integration_guide`).
2. Drop in a token client (`client`) matching your project's conventions.
3. Translate the use-case flow into your code (`recipe`), wiring in your own data.
4. Show you where to set `XBRIDGE_KEY_ID` / `XBRIDGE_SECRET` and how to test against a sandbox key.

It exposes these to your agent:
- **Tools:** `integration_guide`, `recipe({useCase})`, `client({language})`
- **Resources:** `xbridge://rules`, `xbridge://openapi`, `xbridge://recipe/*`, `xbridge://client/*`
- **Prompt:** `integrate`

## Optional: let the agent run calls (execute mode)

For testing in a sandbox, set `BRIDGE_MODE=both` (or `execute`) and provide a key. The agent can then call the live API too. Safety is set by **you at launch**: the environment is fixed by the key (the agent can't switch it), and money-moving tools (disburse, deposit, escrow, collections) only register when you set `BRIDGE_ENABLE_MONEY_MOVERS=true`. Default keeps them off.

```bash
BRIDGE_MODE=both BRIDGE_BAAS_KEY_ID=... BRIDGE_BAAS_SECRET=... \
claude mcp add x-bridge -- node /absolute/path/to/docs/mcp/dist/index.js
```
